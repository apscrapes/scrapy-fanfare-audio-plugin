# scrapy-beep

A Scrapy extension that plays a triumphant fanfare when your spider finishes successfully, and a sad trombone when it fails. No more staring at the terminal waiting to see if the crawl worked.

Both sounds ship with the package â€” no system sounds, no extra dependencies.

## Installation

Install directly from GitHub â€” no PyPI release needed.

**pip**

```bash

```


## Quick start

Add the extension to your project's `settings.py`:

```python
EXTENSIONS = {
    "scrapy_beep.extension.BeepExtension": 500,
}
```

That's it. Run your spider and listen.

## How it works

`BeepExtension` connects to Scrapy's `spider_closed` signal. When the spider finishes, it reads the runtime stats and decides whether the crawl was a success or a failure:

| Condition | Sound |
|---|---|
| `reason == "finished"` **and** at least one item scraped **and** no ERRORs logged | Fanfare |
| Any other reason (`shutdown`, `cancelled`) | Sad trombone |
| `reason == "finished"` but zero items scraped | Sad trombone |
| `reason == "finished"` but ERROR-level log entries present | Sad trombone |

The stats it checks â€” `item_scraped_count` and `log_count/ERROR` â€” are collected automatically by Scrapy's built-in stats system; you don't need to instrument anything.

## Settings reference

| Setting | Type | Default | Description |
|---|---|---|---|
| `BEEP_ENABLED` | `bool` | `True` | Set `False` to disable all audio without removing the extension |
| `BEEP_SUCCESS_SOUND` | `str` | bundled `success.wav` | Path to a WAV file to play on success |
| `BEEP_FAILURE_SOUND` | `str` | bundled `failure.wav` | Path to a WAV file to play on failure |

### Disabling audio

Useful for CI environments or headless servers where you don't want audio playing:

```python
BEEP_ENABLED = False
```

The extension loads cleanly and logs nothing â€” it raises `NotConfigured` internally, which is the standard Scrapy convention for a disabled extension.

### Using custom sounds

Point either setting at any WAV file on disk:

```python
BEEP_SUCCESS_SOUND = "/path/to/my_success.wav"
BEEP_FAILURE_SOUND = "/path/to/my_failure.wav"
```

## Platform support

The extension uses OS-native tools for audio playback â€” no third-party audio library required.

| Platform | Tool | Notes |
|---|---|---|
| macOS | `afplay` | Built into macOS |
| Windows | `winsound.PlaySound` | Python stdlib, no install needed |
| Linux | `aplay` | Part of ALSA utils, ships with most desktop distros |

On headless Linux servers where `aplay` is unavailable, playback fails silently (`check=False`) â€” the spider continues normally and only the sound is skipped.

## Extension priority

The integer in `EXTENSIONS` controls load order relative to other extensions (lower = higher priority). `500` is mid-range and works fine for `scrapy-beep` since it only needs to run after the spider closes. If you have other extensions that also listen to `spider_closed`, order generally doesn't matter â€” Scrapy fires the signal to all listeners.

## Why an extension and not a pipeline?

Pipelines process items one by one as they flow through the crawler. They have no access to final stats and no awareness of how the crawl ended. Extensions hook into spider lifecycle signals â€” `spider_closed` fires once, after everything is done, with the full stats available. Audio feedback on crawl completion is exactly the kind of cross-cutting concern extensions are designed for.
